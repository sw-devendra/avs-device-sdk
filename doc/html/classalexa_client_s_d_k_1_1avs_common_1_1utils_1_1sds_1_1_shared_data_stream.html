Detailed Description

template<typename T>
class alexaClientSDK::avsCommon::utils::sds::SharedDataStream< T >

Class for streaming data from a single producer (Writer) to multiple consumers (Reader). This class implements streaming in a generic manner, and utilizes template traits to decouple from platform specifics related to how memory and state are shared/sychronized between the readers and writers. Care must be taken in choosing appropriate types for the template parameters which will work reliably for the execution environment where the SharedDataStream will be used.

Template Parameters
T::AtomicIndexAn atomic version of Index (see below) which implements the following methods:
  • DefaultConstructible (std::is_default_constructible<AtomicIndex> == true).
  • Basic arithmetic, conversion and assignment operations with Index.
  • load() performs an atomic read of the Index.
  • If the stream will be shared between processes, the AtomicIndex type must be a PODType: (std::is_pod<AtomicIndex> == true).
This should be an equivalent type to Index, but which ensures atomic reads and writes between readers and writers in the execution environment where the SharedDataStream will be used. No methods are called on this type; it must simply be readable and writable with values of type Index or AtomicIndex.
T::AtomicBoolAn atomic boolean type which implements the following methods:
  • DefaultConstructible (std::is_default_constructible<AtomicIndex> == true).
  • Basic conversion and assignment operations with bool.
  • If the stream will be shared between processes, the AtomicBool type must be a PODType: (std::is_pod<AtomicIndex> == true).
This should be an equivalent type to bool, but which ensures atomic reads and writes between readers and writers in the execution environment where the SharedDataStream will be used. No methods are called on this type; it must simply be readable and writable with values of type bool or AtomicBool.
T::BufferA Container type which stores a contiguous block of data and implements the following methods:
  • data() provides access to the underlying storage array.
  • size() returns the size (in bytes) of the buffer.
This type must be capable of sharing data between readers and writers in the execution environment where the SharedDataStream will be used.
T::MutexA Mutex type which implements the following methods:
  • DefaultConstructible (std::is_default_constructible<Mutex> == true).
  • lock() Waits indefinitely for the mutex to unlock and then locks the mutex.
  • unlock Unlocks the mutex.
  • If the stream will be shared between processes, the Mutex type must be a PODType: (std::is_pod<Mutex> == true).
This type must be capable of locking a mutex for readers and writers in the execution environment where the SharedDataStream will be used.
T::ConditionVariableA condition variable type which works with Mutex and implements the following methods:
  • DefaultConstructible (std::is_default_constructible<ConditionVariable> == true).
  • notify_all() unblocks any threads waiting for this condition variable.
  • wait(lock) waits indefinitely for lock.
  • wait(lock, predicate) waits indefinitely using lock for the predicate to be satisfied.
  • wait_for(lock, timeout, predicate) waits using lock up to the specified timeout for the predicate to be satisfied.
  • If the stream will be shared between processes, the ConditionVariable type must be a PODType: (std::is_pod<ConditionVariable> == true).
This type must be capable of synchronizing between readers and writers in the execution environment where the SharedDataStream will be used.
T::traitsNameA unique string value which describes the collection of traits specified by T. This string is used to ensure that a SharedDataStream attempting to open() a buffer is using the same set of traits that were originally used to create() the buffer.

This function calculates the buffer size needed to support a SharedDataStream with the specified parameters. This function can be safely called from multiple threads or processes.

Parameters
nWordsThe number of data words the stream will be able to hold.
wordSizeThe size (in bytes) of words in the stream. All SharedDataStream operations that work with data or position in the stream are quantified in words. The stream's data storage capacity in bytes is nWords * wordSize. This parameter defaults to 1.
maxReadersThe maximum number of readers the stream will support. This parameter defaults to 1.
Returns
The buffer size (in bytes) required to support the specified parameters, or zero if parameters are invalid.

This function creates a new SharedDataStream. It will first verify that the Buffer is large enough to hold the stream's header, and will then proceed to initialize the header fields. This function must not be called more than once on buffer. This function must return before attempting to open() buffer with another SharedDataStream.

Parameters
bufferThe Buffer which this stream will use to store its header and stream data. Existing contents of buffer will be overwritten by this function. Note that this function will fail if buffer is not large enough to hold the header or is not a multiple of wordSize (see calculateBufferSize()).
wordSizeThe size (in bytes) of words in the stream. All SharedDataStream operations that work with data or position in the stream are quantified in words. This parameter defaults to 1.
maxReadersThe maximum number of readers the stream will support. This parameter defaults to 1.
Returns
The new stream if buffer was successfully initialized, else nullptr.

template<typename T >
std::unique_ptr< typename SharedDataStream< T >::Reader > alexaClientSDK::avsCommon::utils::sds::SharedDataStream< T >::createReader ( size_t  id,
typename Reader::Policy  policy,
bool  startWithNewData = false,
bool  forceReplacement = false 
)

This function adds a Reader to the stream using a specific id. This function can be safely called from multiple threads or processes.

A SharedDataStream maintains a fixed-size array of getMaxReaders() Readers. If addReader() is called without specifying an id, it will return the first unused Reader in the array and assign the corresponding id. If an application wants to directly manage Reader ids, it can instead call addReader() with a specific id (any value in the range [0, getMaxReaders())), and this function will add the Reader with that id if it is not already in use.

Parameters
idThe id to use for this Reader.
policyThe policy to use for reading from the stream.
startWithNewDataFlag indicating that this Reader should start reading data which is written to the buffer after this Reader is created. If this parameter is set to false, the Reader will start with the oldest valid data in the buffer. This parameter defaults to false.
forceReplacementIf set to true, this parameter forcefully deallocates the Reader before allocating a new one. This parameter can be used when a Reader is known to be unused (possibly due to a process crash), but was not cleanly destroyed. This parameter is set to false by default.
Returns
nullptr if the requested Reader id is already allocated, else the new Reader.
Warning
Calling this function with forceReplacement = true will allow the call to createReader() to succeed, but will not prevent an previously existing Reader from reading from the stream. The forceReplacement = true option should only be used when higher level software can guarantee that the previous Reader will no longer be used to attempt to read from the stream.
